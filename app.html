Staging Review

Coins saved to Staging already hold a reserved CollectionID (assigned at save time, not deferred). Promote moves a row into the collection with that same ID carried forward unchanged; Reject removes it and frees its ID. IDs never backfill — only the highest reserved ID becomes reusable when rejected; a rejected mid-sequence ID stays a permanent gap (deliberate, to avoid ever reusing an ID a photo/receipt may already reference).
